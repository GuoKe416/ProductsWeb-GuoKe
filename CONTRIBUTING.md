# 贡献指南

感谢您考虑为商品文件库管理系统做出贡献！以下是一些指导方针和步骤，帮助您更好地参与到项目中来。

## 报告问题

如果您发现任何问题或有功能请求，请通过GitHub Issues提交，确保包含：

1. 清晰的问题描述
2. 重现步骤（如适用）
3. 预期行为与实际行为
4. 您使用的环境详情（PHP版本、MySQL版本、浏览器等）
5. 可能的解决方案（如有）

## 提交代码

1. Fork仓库
2. 创建您的特性分支 (`git checkout -b feature/amazing-feature`)
3. 提交您的更改 (`git commit -m '添加一些功能'`)
4. 推送到分支 (`git push origin feature/amazing-feature`)
5. 打开Pull Request

## 代码风格

- 遵循PSR-4自动加载规范
- 使用有意义的变量名和函数名
- 添加必要的注释以解释代码的功能和目的
- 保持代码的简洁性和可读性

## 开发环境设置

1. 克隆仓库到本地
2. 创建测试数据库
3. 运行`install.php`完成安装

## 测试

在提交代码之前，请确保：

1. 功能按预期工作
2. 不引入新的错误
3. 代码兼容PHP 7.0+和MySQL 5.6+

## 安全考虑

请特别注意以下安全实践：

1. 所有用户输入必须经过验证和净化
2. SQL查询必须使用参数化方式以防止SQL注入
3. 密码必须加密存储
4. 避免将敏感信息暴露在前端

## 文档

如果您的贡献涉及功能变更或新功能添加，请更新相应的文档，包括：

1. 更新README.md中的功能描述
2. 添加任何新的配置选项或使用说明
3. 如适用，更新安装说明

## 许可证

通过提交代码，您同意您的贡献将在MIT许可证下发布。
